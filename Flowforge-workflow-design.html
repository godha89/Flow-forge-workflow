🧩 Dynamic Engine

Workflows are loaded from JSON/YAML definitions at runtime. No code deploy required for new workflow types. The engine reads a definition, resolves checks and steps, and executes them via the registry.

Runtime Loading Hot Reload Zero Deploy
🔌 Plug & Play Checks

Each eligibility check is a self-contained unit registered by key. Workflow definitions reference check keys — the engine resolves and runs them. Add new checks without touching existing code.

Check Registry Composable Ordered
📡 Full Observability

Every state transition, check result, and process call emits structured events. OpenTelemetry spans trace end-to-end. Each workflow instance has a full audit trail queryable by ID.

Eligibility Engine
Generic, plug-and-play checks mapped per workflow type
Available Check Registry
🪪

KYC_VERIFIED

Validates customer KYC status against identity service. Fails if not verified or expired.

type: external_api cacheable: true ttl: 3600s
💰

BALANCE_SUFFICIENT

Checks account balance meets minimum threshold. Threshold is a configurable parameter.

type: rule param: min_balance async: false
📊

CREDIT_SCORE_THRESHOLD

Pulls credit score and evaluates against configurable band. Supports min/max range.

type: external_api param: min_score param: max_score
🚫

SANCTIONS_CLEAR

Screens entity against OFAC / sanctions lists. Hard fail — no override allowed.

type: external_api override: false hard_fail
⚠️

RISK_SCORE_LOW

Evaluates internal risk model output. Configurable threshold, supports soft override.

type: ml_model override: true param: threshold
📋

DOCUMENT_COMPLETE

Validates all required documents are present and unexpired for a given workflow type.

type: rule param: required_docs[] cacheable: false

Sync & Async Process Model
REST for sync operations · RabbitMQ for async operations
Sync — REST Process
REST Executor
SYNC

→
HTTP Call
→

→
Map to Context
Blocks workflow execution until response. Supports timeout, retry with exponential backoff, circuit breaker. Response is mapped to workflow context for downstream steps.
Timeout Config Auto Retry Circuit Breaker Response Mapping
``` processes: - id: fraud_check type: REST method: POST endpoint: /api/fraud/evaluate timeout_ms: 3000 retry: max_attempts: 2 backoff: fixed_200ms circuit_breaker: threshold: 0.5 window_ms: 60000 response_map: ctx.fraud_score: $.score ctx.fraud_flags: $.flags

RabbitMQ Executor
ASYNC

→

→

→
Resume WF
Publishes message to a RabbitMQ exchange, checkpoints workflow state, and suspends. A dedicated reply consumer listens on the reply queue and resumes the workflow when the response arrives, matched by correlation ID.
